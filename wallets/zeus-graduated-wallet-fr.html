Comment commencer à utiliser Zeus comme portefeuille LN progressif, en tant que nouvel utilisateur, étape par étape

| | | | | | | | |

Mise à jour le 15 décembre 2025

Introduction

OK, j'ai constaté une forte demande pour ce type de guide. Beaucoup de personnes m'ont contacté pour savoir comment débuter avec le Lightning Network sans utiliser les canaux LN, pour diverses raisons.

Je suis sidéré de voir que certains ne savent toujours pas utiliser Zeus, ce portefeuille mobile Lightning Network très performant et pur, sans fioritures ni fonctionnalités cachées, avec de nombreuses options. Vous trouverez ici de nombreux liens vers d'autres guides que je vous recommande de consulter.

Ce guide s'adresse à tous les nouveaux utilisateurs de Bitcoin qui ne maîtrisent pas encore les termes liés aux nœuds LN, à la liquidité, aux swaps, etc., qui sont encore peu connus. Je vais essayer d'être aussi clair que possible, mais veuillez suivre attentivement toutes les instructions.

Qu'est-ce qu'un « portefeuille progressif » ? Comme Evan Kaloudis l'a présenté à la conférence BTC++ (voir la vidéo complète ci-dessous), il s'agit d'un portefeuille avec lequel vous commencez par le bas et progressez jusqu'au sommet, en accumulant lentement des satoshis et en ouvrant des canaux lorsque cela est nécessaire et que vous vous sentez à l'aise pour franchir une nouvelle étape, jusqu'au sommet de l'échelle des connaissances, au niveau le plus élevé, lorsque vous exécutez un nœud distant puissant géré par Zeus.

Alors, commençons… mes jeunes padawans…

Ce guide du portefeuille progressif comporte 3 niveaux principaux :

  • Étape 1 - Mortel - Commencez par les bases avec un portefeuille Cashu et/ou des comptes NWC et Lndhub
  • Étape 2 - Dieu - Passez au nœud intégré auto-dépositaire Zeus et ouvrez des canaux LN
  • Étape 3 - Olympien - Atteignez le niveau le plus élevé, en exécutant votre propre nœud distant géré par Zeus.

ÉTAPE 1 - NIVEAU MORTAL

Principales caractéristiques de ce niveau :

  • Intégration avec un service de garde ou de confiance
  • Adresse Lightning
  • Gestion des canaux Lightning Network non disponible
  • L'utilisateur peut encore découvrir le fonctionnement de Lightning Network par moments.
  • Le portefeuille invite l'utilisateur à passer à la garde autonome à mesure que son solde augmente.

Option A - PORTEFEUILLE CASHU

Vous êtes un nouvel utilisateur Bitcoin et vous n'avez aucun satoshi, ni aucun moyen d'ouvrir un canal LN.

Tout d'abord, je vous recommande de lire mes guides pour débutants sur Bitcoin afin d'acquérir des connaissances de base sur les termes et concepts liés au Bitcoin, tels que les portefeuilles on-chain, les adresses, les canaux LN, etc. - Guides pour débutants Bitcoin de DarthCoin

Vous connaissez maintenant les bases du Bitcoin ; nous n'entrerons pas dans les détails ici. Vous souhaitez maintenant recevoir et accumuler une certaine quantité de satoshis, puis les transférer vers un portefeuille on-chain, un portefeuille froid ou tout autre emplacement de votre choix. Beaucoup de personnes achètent des satoshis sur des plateformes d'échange, mais ne savent pas comment les retirer via LN.

Qu'est-ce que Cashu ? C'est une question que se posent souvent les nouveaux utilisateurs…

Cashu est un protocole de paiement électronique Chaumian gratuit et open source, basé sur Bitcoin et offrant de bonnes propriétés de confidentialité. C'est plutôt comme un jeton contenant des satoshis, comme une carte cadeau chargée de satoshis, mais sécurisé et privé.

Attention cependant, il s'agit toujours d'un service de dépôt, ce qui signifie qu'il existe un risque que le dépositaire sous-jacent (l'établissement de monnaie) vole vos fonds.

Plus d'informations sur Cashu ici : https://cashu.space

Zeus propose une solution simple et efficace : un portefeuille Cashu intégré. Pas de canaux Lightning Network, sauvegarde de la phrase de récupération, réception immédiate sans frais cachés, swaps intégrés à la blockchain, interface utilisateur conviviale, plusieurs établissements de monnaie et… un excellent support utilisateur de l'équipe Zeus.

  • Une fois que vous avez reçu une certaine quantité de sats que vous souhaitez transférer vers vos propres portefeuilles de conservation sur la blockchain, utilisez le service d'échange intégré de Zeus (avec plusieurs fournisseurs d'échange) et transférez les sats hors de Cashu.

  • Vous n'avez pas besoin d'ouvrir un canal LN pour effectuer un swap sur la blockchain. Vous pouvez effectuer le swap directement depuis votre portefeuille Cashu.
  • Facultatif : vous pouvez créer votre adresse LN si vous souhaitez l'utiliser sur Nostr pour les zaps.

Il sera nécessaire de mettre votre Zeus en mode persistant.

Pendant cette période, Zeus vous indiquera le moment optimal pour transférer votre solde Cashu et quand vous serez prêt à ouvrir votre premier canal LN.

RAPPEL : N'oubliez pas de sauvegarder votre phrase de récupération !

Le portefeuille Cashu est lié à la phrase de récupération du nœud intégré Zeus. Par conséquent, même si vous n'utilisez pas le nœud intégré avec les canaux, il est indispensable de générer une sauvegarde de votre portefeuille Cashu. Vous pouvez toujours utiliser le nœud intégré Zeus sur la blockchain si vous le souhaitez. Aucun problème à l'utiliser comme zone tampon, jusqu'à ce que vous transfériez une plus grande quantité de satoshis vers un autre portefeuille froid.

Option B - Comptes NWC / lndhub

Si vous ne souhaitez pas utiliser le portefeuille Cashu ou si vous voulez simplement une autre façon d'utiliser Zeus sans les canaux LN, vous pouvez utiliser n'importe quel fournisseur NWC ou compte lndhub. Il s'agit généralement de comptes de dépôt, mais si vous avez un ami qui gère une banque LN pour votre communauté locale, c'est une autre solution simple pour utiliser Zeus.

Ces comptes n'ont pas besoin d'exécuter le nœud intégré ! Ils sont totalement indépendants et vous pouvez en ouvrir autant que vous le souhaitez.

Lorsque vous utilisez Zeus pour la première fois, vous pouvez créer directement un compte et sélectionner l'interface NWC (Nostr Wallet Connect) ou lndhub, au lieu d'un nœud intégré ou d'un nœud distant.

Il vous sera demandé d'insérer la chaîne NWC ou l'URL lndhub des comptes que vous souhaitez utiliser avec Zeus.

Pour savoir où obtenir un compte NWC, veuillez consulter https://nwc.dev et lire la suite.

Personnellement, je préfère utiliser un nœud Rizful jetable et configurer un nœud cloud simple, avec un seul canal.

Configurer votre nœud Rizful :

  • Rendez-vous sur la page Rizful Node in the cloud et créez un compte anonyme gratuit.
    • Sauvegardez toutes les informations obtenues lors de la création de votre nœud LN : ID du nœud, adresse LN (personnalisable), identifiants de connexion, et les deux chaînes NWC (réception uniquement et envoi/réception). Vous trouverez une documentation complète ici.
  • Achetez votre premier canal entrant. Leur documentation est très utile et claire.

  • Cliquez sur le bouton « Obtenir un canal entrant » et sélectionnez la taille de canal souhaitée. Choisissez la bonne taille ! Estimez la quantité de données que vous recevrez sur une période donnée, en fonction de votre activité. Par exemple, si vous prévoyez de recevoir environ 500 000 satoshis par semaine. Vous achèterez donc un canal de 500 000 satoshis pour environ 13 000 satoshis de frais. Le canal sera ouvert et prêt à l'emploi presque instantanément.

    Vous pouvez également ouvrir un canal avec d'autres nœuds Lightning Network, en utilisant les fonds de votre portefeuille on-chain. Pour cela, vous devrez d'abord effectuer un dépôt sur votre nœud Rizful on-chain, puis ouvrir le canal. Attention : il s'agira d'une « capacité sortante » (https://rizful.com/docs/balances-and-capacities#outbound-capacity). Vous ne pourrez donc pas recevoir de fonds tant que vous n'aurez pas effectué de paiements et consommé une partie du canal. Un canal Lightning Network est comme un verre rempli d'eau : il peut être plein ou vide, selon vos besoins. Mais en tant que commerçant, il vaut mieux commencer avec un verre vide, car vous recevrez davantage par la suite.

  • Générez vos chaînes NWC. Vous pouvez utiliser NWC « réception uniquement » ou un accès complet avec « envoi/réception », selon votre utilisation dans Zeus.
  • Sur le premier écran de Zeus, sélectionnez « Avancé » et créez une nouvelle connexion en utilisant l'option NWC (Nostr Wallet Connect) du menu déroulant. Inscrivez un nom sur votre portefeuille et, si vous le souhaitez, une image.

  • Collez la chaîne NWC « receive-only » ou « send/receive » du nœud Rizful dans le champ URL NWC et enregistrez la configuration du portefeuille.

Et voilà ! Vous disposez désormais d'un portefeuille LN pleinement fonctionnel sur votre Zeus, utilisant un nœud LN distant via une connexion NWC. Vous pouvez accumuler sur ce canal entrant Rizful tous les sats souhaités et, lorsque vous le souhaitez, les transférer vers l'un de vos portefeuilles onchain.

Cette option avec un nœud Rizful est également pratique pour créer un leurre, mais elle permet aussi de disposer d'un nœud LN disponible 24 h/24 et 7 j/7, sans avoir à synchroniser quoi que ce soit sur votre Zeus. Elle est particulièrement recommandée aux utilisateurs situés dans des zones reculées où la synchronisation des neutrinos pose problème. J'ai expliqué ces aspects dans un autre guide ici.

Ainsi, avec un compte NWC ou lndhub, vous pouvez éviter ce genre de situations.

N'oubliez pas que ces comptes sont toujours sous séquestre !

ÉCHANGES VERS LA CHAÎNE

Nous supposons que vous avez déjà reçu suffisamment de sats sur vos comptes Cashu ou NWC et que vous souhaitez maintenant les transférer de ces comptes sous séquestre vers vos propres portefeuilles sur la chaîne, pour la mise en cache, la conservation ou les réserves à long terme.

La procédure est très simple : commencez par préparer vos portefeuilles sur la chaîne :

  • Vous pouvez utiliser le portefeuille sur la chaîne du nœud Zeus intégré.
  • Vous pouvez utiliser n'importe quel portefeuille sur la chaîne externe, de préférence une application de bureau comme Sparrow, Electrum ou Bitcoin-Safe pour une gestion optimale et sécurisée de vos UTXO. Portefeuilles.

Veuillez lire mon guide précédent sur comment stocker vos Bitcoins sur trois niveaux : dépenses, cache/zone tampon et portefeuilles froids/coffres-forts. Vous y trouverez également un tableau d'exemples de tailles recommandées pour les UTXO dans vos portefeuilles onchain. Veuillez lire ce guide, il est très important.

Ensuite, allez dans Zeus, menu en haut à gauche, puis Swaps.

Il est préférable de sauvegarder les clés d'échange uniques fournies par Zeus, car elles seront nécessaires en cas d'échec de l'échange. Pour cela, rendez-vous en haut à droite et cliquez sur la clé. Enregistrez-la en lieu sûr, hors de votre appareil, par exemple dans votre gestionnaire de mots de passe.

Vous trouverez également en haut à droite une roue d'options permettant de choisir un autre fournisseur d'échange. Zeus est le fournisseur par défaut, mais d'autres sont disponibles, comme Boltz, SwapMarket, Eldamar ou un fournisseur personnalisé.

Comme vous pouvez le constater, Zeus offre un large choix et n'est pas un système fermé comme d'autres. Tout est transparent.

Vous êtes ensuite invité à sélectionner le type d'échange souhaité. Par défaut, il s'agit de recharger un canal LN existant, mais vous pouvez changer d'échange en appuyant sur la flèche entre « onchain » et « LN ».

Vous verrez qu'il vous est maintenant demandé de fournir une adresse onchain de destination. Il peut s'agir de n'importe lequel de vos portefeuilles onchain internes ou externes.

Saisissez le montant que vous souhaitez échanger. Je vous suggère de ne pas utiliser la totalité de vos sats accumulés ; conservez environ 1 % pour les frais et les réserves éventuels. Vous serez ensuite invité à payer une facture LN, que vous pouvez régler avec votre compte Zeus Cashu ou NWC. L'échange sera effectué en quelques instants. Vos sats seront confirmés à l'adresse onchain que vous avez indiquée.

Vous êtes maintenant prêt à répéter l'accumulation et à effectuer un nouvel échange.

GÉRER VOS PAIRS NEUTRINO ET UN NŒUD EN BON FONCTIONNEMENT

N'oubliez pas : Zeus est un nœud LND ! Ce n'est pas un simple portefeuille ou un compte de dépôt.

Cela signifie qu'il doit être parfaitement synchronisé avec la chaîne et le graphe LN pour pouvoir être utilisé correctement. Dans les guides de ressources, j'ai expliqué plus en détail comment gérer les pairs neutrino. Veuillez les lire.

Ici, je donnerai plus de conseils sur la façon de maintenir votre nœud Zeus en bon état, car je vois encore des personnes qui ne comprennent pas certains aspects.

  • Aucun bloc synchronisé = le nœud LN ne démarre pas = aucun canal en ligne = aucun solde utilisable sur la blockchain
  • Zeus utilise BIP157 / Neutrino pour synchroniser les en-têtes des blocs, avec des filtres de blocs compacts.
  • En savoir plus sur les filtres de blocs compacts ici
  • Un bon article sur la confidentialité de Neutrino ici
  • Il est inutile d'utiliser Zeus avec Tor pour la synchronisation. Tor ne fait qu'augmenter la latence et peut interrompre la synchronisation. Même un VPN n'est pas nécessaire, car cela pourrait également augmenter la latence.
  • La synchronisation Neutrino nécessite un bon ping (faible latence) pour être rapide. Avoir une connexion internet haut débit locale pour le téléchargement ne sert à rien si votre ping vers le pair Neutrino est faible. N'oubliez pas que le ping est déterminé par la distance entre votre emplacement et le pair. Ajouter des couches réseau et des points de connexion supplémentaires augmentera la latence, ce qui entraînera un échec de la synchronisation.
  • Si vous gérez un nœud Bitcoin ou si un ami local en gère un, vous pouvez vous y connecter pour bénéficier d'un meilleur ping. Voici un guide à ce sujet.
  • Avant d'utiliser le nœud Zeus, configurez toujours vos pairs Neutrino en conséquence. Consultez la FAQ de la documentation Zeus ou mes guides. La procédure y est très bien expliquée.
  • Neutrino et la recherche de chemin côté client (synchronisation Express Graph en local) offrent une excellente confidentialité pour exécuter un nœud Lightning Network privé. Ne compliquez pas les choses avec Tor. Tor n'offre aucune protection supplémentaire en matière de confidentialité, et ne fait qu'empirer les choses.
  • Recherche de chemin côté client : Zeus construit les chemins de paiement Lightning des utilisateurs sur leurs appareils. Nous ne connaissons donc pas la destination finale des paiements de nos utilisateurs, même lorsqu'ils transitent par nos canaux. Non seulement nous ne voulons pas savoir comment vous utilisez votre argent, mais nous concevons notre système de manière à ce que cela soit impossible. Tous les portefeuilles ne peuvent pas en dire autant.

Autre point important : si vous utilisez fréquemment Zeus au cours de la journée, vous pouvez le passer en « mode persistant ». Le service LND fonctionnera alors en arrière-plan et maintiendra la synchronisation active, prêt à effectuer et recevoir des paiements à tout moment. Ceci est également nécessaire si vous souhaitez utiliser Zeus avec les nouveaux sous-comptes NWC. C'est comme maintenir un nœud LN actif, sauf que celui-ci est sur votre mobile.

Si vous ne l'utilisez pas fréquemment, mais que vous souhaitez tout de même qu'il soit prêt, il est préférable de le synchroniser presque quotidiennement, même si vous n'effectuez aucun paiement.

Voici une autre astuce : si vous n'effectuez pas souvent de paiements, mais que vous souhaitez une synchronisation optimale, activez dans le nœud intégré (Express Graph Sync) les options « Activer EGS » et « Réinitialiser EGS au démarrage ». Ces deux options maintiendront le graphique LN à jour, vous offrant ainsi les meilleurs chemins de paiement, construits localement par votre propre nœud, et non par un LSP avec lequel vous avez des canaux.

Autre astuce : comment préserver la qualité de vos données Zeus.

Avec le temps, le fichier de base de données des canaux grossit, car plus vous l'utilisez, plus les mises à jour sont fréquentes. Il est donc recommandé de compacter cette base de données.

Accédez au nœud intégré, puis à la section « Dépannage » et activez l'option « Compacter la base de données ». Un redémarrage vous sera demandé. Procédez et patientez quelques instants.

Vous pourrez ensuite consulter les informations de l'application pour voir la quantité de données réduites. Il est conseillé d'effectuer cette opération chaque semaine ou chaque mois, et non quotidiennement. Réduire la taille de la base de données contribue également à économiser la batterie.

Une autre astuce, également disponible dans la section « Dépannage », consiste à utiliser l'option « Supprimer les fichiers Neutrino ». Utilisez cette option UNIQUEMENT si vous constatez un comportement anormal lors de la synchronisation, même si votre ping avec vos pairs est bon. Parfois, si vous ne synchronisez pas pendant une longue période, ces fichiers peuvent se corrompre ou devenir obsolètes. Il est alors préférable de les supprimer et de relancer la synchronisation. En général, ils font environ 400 à 500 Mo (il s'agit uniquement des en-têtes des blocs). Vous pouvez les supprimer sans risque grâce à l'option disponible dans la section de dépannage, puis redémarrer Zeus. Soyez patient ! La synchronisation de votre nœud recommencera à zéro et le portefeuille sera également analysé à nouveau. Il est donc conseillé de le mettre en mode persistant et d'attendre. Avec un bon pair Neutrino, cela ne prend généralement pas plus de 5 à 10 minutes.

ÉTAPE TROIS - NIVEAU OLYMPIEN

EXÉCUTER VOTRE NŒUD À DISTANCE AVEC ZEUS

Principaux aspects de ce niveau :

  • Vous êtes désormais un administrateur de nœud autonome.
  • Le nœud LN peut être installé chez vous ou hébergé à distance.
  • Maîtrise complète de la liquidité des canaux.
  • Gérez vos propres canaux LN et votre liquidité.

Il est temps de passer aux choses sérieuses. Ce niveau est destiné aux utilisateurs avancés, ayant déjà accumulé suffisamment de satoshis et une bonne connaissance de la gestion de la liquidité LN.

Vous commencez par installer une machine dédiée au nœud chez vous ou dans le cloud sur un serveur VPS, puis vous y connectez votre Zeus via les connexions spécifiques. En savoir plus ici.

J'ai également rédigé plusieurs guides sur l'exécution de nœuds distants. Je souhaite simplement vous rappeler quelques points :

  • Il existe deux types de nœuds : les nœuds de routage publics et les nœuds privés (non routés). Choisissez judicieusement. Pour un usage personnel avec un trafic important, un nœud privé suffit. Si vous souhaitez devenir routeur, vous aurez besoin d'un nœud public, mais avec un budget conséquent et une grande fiabilité.
  • Vous pouvez exécuter un nœud complet avec synchronisation de toute la blockchain Bitcoin, mais vous pouvez également exécuter un nœud LND sur Neutrino. Ce dernier peut être exécuté très facilement sur n'importe quel VPS bon marché, car il ne consomme pas autant d'espace qu'un nœud Bitcoin complet.
  • Si vous exécutez un nœud de routage public, veuillez NE PAS l'exécuter sur un Raspberry Pi bas de gamme derrière Tor. Les nœuds publics DOIVENT être extrêmement fiables et connectés au réseau non chiffré pour des performances optimales.
    • Si vous ne pouvez pas utiliser le réseau classique avec un nœud domestique, nous vous recommandons de consulter cet excellent guide expliquant comment créer un tunnel VPN WireGuard avec n'importe quel VPS.
  • Vous trouverez ici plusieurs guides concernant les nœuds distants, la gestion des canaux, les frais, les pairs, les outils, etc.. Veuillez les consulter.
