Quais soluções Bitcoin LN você pode começar a usar como um pequeno comerciante

| | | | | | | | |

Postado originalmente em Substack em 26 de outubro de 2021. Atualizado aqui em 24 de fevereiro de 2025.

Muitas vezes ouvi esta pergunta: "Como começar a aceitar Bitcoin, como um pequeno comerciante?"

E aqui eu gostaria de apresentar algumas soluções simples e rápidas, campo de testes, pequenos passos simples, para novos usuários que ainda não podem/não querem executar uma máquina de nó LN completa.

Metas a serem alcançadas:

  • O comerciante quer começar a aceitar BTC através do LN, com passos simples, apenas o suficiente para testá-lo, aprender como funciona e ser capaz de receber BTC.
  • Menor custo possível
  • Acomode-se com os termos do Bitcoin e da Lightning Network, como usar carteiras, códigos QR, faturas LN, LNURL, taxas etc.
  • Uma maneira de mover os fundos recebidos para uma carteira "hodling"
  • Prepare a mentalidade para a próxima etapa: executar um nó LN de comerciante. essas soluções são apenas o primeiro passo, temporário, até que o comerciante esteja pronto para ter um nó completo, seu próprio banco para gerenciar seu próprio dinheiro.
  • Ter acesso a um plugin WP Woocommerce para integrar rapidamente com lojas virtuais de e-commerce existentes
  • Esta NÃO é uma solução para longo prazo, é apenas para prepará-lo e entender como o Bitcoin LN funciona para uma pequena loja. Para uma solução de longo prazo e autocustódia, consulte a seção Soluções avançadas abaixo.

Soluções disponíveis:

Cada um tem suas próprias particularidades, recursos, nível de complexidade e casos de uso. Sugiro começar a usar 2-3 deles e se sentir confortável, enquanto estuda os outros.

  • CoinOS - custodial, integração muito simples, NWC, nenhum aplicativo necessário
  • Walletano - custodial, integração muito simples, nenhum aplicativo necessário
  • IBEX Pay - custodial, painel de comerciante complexo, vários recursos, conversão fiduciária, plugin Woocommerce
  • Blink Wallet - custodial, endereço LN, PoS, API
  • Opago Pay - não custodial, PoS offline simples, conversão fiduciária, plugin Woocommerce
  • Swiss Bitcoin Pay - custodial, aplicativo simples, aplicativo PoS, plugin Woocommerce
  • Pague com Flash - Aplicativos simples e independentes, aplicativos PoS, plug-ins de plataforma de comércio eletrônico
  • Pague com Flash - aplicativo simples e autocustodial, aplicativo PoS, plugins de plataformas de e-commerce
  • OpenNode - processador de pagamento de custódia, plugin Woocommerce
  • CoinGate - processador de pagamento de custódia, plugin Woocommerce
  • Sats Mobi - custódia, bot Telegram LN com opções para PoS
  • LN Voltz Bank - custódia, instância LNBits para seu próprio uso, plugin Woocommerce
  • Alby - autocustódia, solução poderosa com vários recursos, NWC, plugin Woocommerce
  • Bluewallet - carteira LNdhub de custódia ou nó LN LDK incorporado de autocustódia
  • Phoenix / Green / Breez / Carteira móvel Electrum - nós LN de autocustódia móveis simples
  • Blixt - nó LND de autocustódia completo no seu celular, endereço LN
  • Zeus - nó LND móvel de autocustódia completo, gerenciamento de nó remoto, contas LNDHUB, endereço LN, PoS
  • Zaprite - Solução poderosa com vários métodos de pagamento mistos
  • Soluções avançadas Execute seu próprio nó BTC LN como um comerciante (servidor BTCPay ou LNBits)

Um caso simples em particular também poderia ser usar Lightning Address. Para freelancers ou faturas ocasionais pagas em BTC, você pode configurar um LN Address com um servidor ou serviço federado ou até mesmo com seu próprio domínio, vinculado ao seu próprio nó. Aqui está um guia.

1 - CoinOS #

Descrição / recursos

  • https://coinos.io/
  • interface da web, PWA, fácil de usar também em dispositivos móveis, NWC (Nostr Wallet Connect)
  • Custodial, não KYC
  • não precisa instalar nada, basta criar uma conta/carteira anônima
  • troca perfeita entre onchain/LN
  • controle total das chaves da carteira
  • suporte total para LN Address, agora você pode ter um como username@coinos.io
  • suporte total para LNURL enviar/receber/login
  • suporte ecash

Cenário de caso de uso

  1. abrir/criar uma conta e salvar o usuário/senha, 2FA, chaves, LNURL-auth
  2. Na conta (canto superior direito) você verá que foi criado um endereço exclusivo para sua carteira, para fácil acesso, como https://coinos.io/user-name
  3. para começar a receber pagamentos, basta clicar no botão receber. Por padrão, será a fatura vazia do Lightning. Você terá que adicionar um valor, algumas carteiras LN ainda não são capazes de pagar uma fatura LN vazia.
  4. Você também pode usar o LNURL fornecido para receber pagamentos sem fatura LN, para que você possa imprimir o código QR LNURL em um papel e apresentá-lo aos clientes, sem precisar abrir a carteira online. Uma vez que o cliente mostrará o pagamento feito, não há como reverter o pagamento, então não há necessidade de mais confirmações.
  5. há também a opção de receber no Lightning Address, muitas carteiras LN são capazes de enviar para username@coinos.io e você não precisa fazer nenhuma fatura, o cliente apenas envia o valor necessário.
  6. No final do dia ou da semana, ou quando quiser, o comerciante pode sacar os fundos da carteira CoinOS para qualquer outra carteira pessoal. A troca de fundos LN para o endereço onchain é feita automaticamente (envolve uma taxa de troca). Mas também pode ser enviada para outra carteira LN, sem nenhuma outra taxa além da taxa de tx (extremamente pequena).
  7. Se você quiser ter um ícone simples na área de trabalho do telefone, como um aplicativo, abra https://coinos.io/username no navegador do seu celular (eu testei com o Firefox no Android), vá ao menu e clique em "Instalar". Ele criará um aplicativo de ícone simples que você pode arrastar para o widget da área de trabalho do seu telefone e abri-lo na próxima vez como um aplicativo.

10 - Sats Mobi

Descrição / recursos

  • https://t.me/satsmobi
  • Carteira de bot Telegram LN, pode ser usada no aplicativo Telegram e também importada para Zeus, BitBanana ou Bluewallet como carteira LN normal
  • Custodial, não KYC
  • esta carteira é custodial, então deve ser usada apenas para pequenas quantias temporárias e retiradas para uma carteira pessoal, periodicamente
  • suporte para LNURL e Lightning Address (TGuser@sats.mobi), fácil de usar no modo offline
  • Também pode ser usada como um PoS simples
  • taxas muito pequenas para txs, sem taxas para abrir/manter a carteira da conta
  • Compre BTC em app

Cenário de caso de uso

  1. o comerciante abrirá/criará um nome de usuário do Telegram
  2. automaticamente terá disponível um LNURL em https://sats.mobi/@TG-user-name que pode receber pagamentos LN imediatamente, sem necessidade de fatura LN, mínimo de 100 sats
  3. automaticamente terá disponível um Lightning Address como username@ln.tips onde os clientes podem pagar diretamente com carteiras que o suportam.
  4. o comerciante pode criar também faturas LN, no Telegram em chat direto com @SatsMobiBot com o comando /invoice {amount sats} {optional description} e mostrar o código QR ou da fatura LN para o cliente
  5. o comerciante também pode importar a carteira SatsMobiBot para o aplicativo Zeus / Bluewallet e usá-la como uma carteira LN normal e criar faturas para seus clientes, mover fundos para outras carteiras, trocar para onchain (usando serviços de swap)

11 - LN Voltz

Descrição / recursos

  • LN Voltz Bank
  • É uma instância pública do LNbits, fácil de criar contas de custódia, caso você precise de um LNbits rápido, com todos os seus recursos.
  • Custodial, não KYC. Esta carteira é custodial, portanto, deve ser usada apenas para pequenas quantias temporárias e retiradas para uma carteira pessoal, periodicamente
  • suporte para LNURL e Lightning Address, PoS, PoS offline, várias extensões úteis, plugin Woocommerce para LNbits
  • Aqui está um guia completo sobre como usar LNbits para comerciantes
  • taxas muito pequenas para txs, sem taxas para abrir/manter a carteira da conta

Cenário de caso de uso

  1. O comerciante abrirá/criará uma conta Voltz, gratuitamente
  2. Pode conectar sua loja de comércio eletrônico diretamente a esta instância LNbits, por meio do Woocommerce Plugin LNbits
  3. Primeiro você precisará adicionar as extensões necessárias: PayLinks, SatsPay Server, LNDHub, Scrub, Onchain, Boltz swap, TPOS e outras, se necessário.
  4. Então crie um link SatsPay para poder gerenciá-lo a partir das cobranças do plugin Woocommerce em sua loja virtual.
  5. Adicione um xpub onchain de qualquer uma de suas carteiras externas, pode ser fria ou quente. Isso será usado para sacar os sats diretamente para sua carteira externa de autocustódia.
  6. Na extensão PayLinks você pode criar endereços LNURL e LN ilimitados que desejar, usados ​​para códigos de pagamento QR estáticos diretos ou com valor específico.
  7. A extensão Scrub pode ser usada se você quiser sacar automaticamente os sats cobrados em sua loja virtual desses LNbits de custódia. Opcionalmente, você pode usar a extensão Boltz para trocar os sats do LN para qualquer endereço onchain.
  8. Com a extensão LNdhub, você pode gerenciar o acesso às suas contas (você pode criar várias contas de carteira neste caso) usando um aplicativo externo como Zeus, Bluewallet, BitBanana.

14 - Phoenix, Green, Breez ou Electrum #

Descrição / recursos

  • Links: Phoenix | Green | Breez | Electrum
  • carteiras móveis simples que operam onchain/LN na mesma interface
  • estão abrindo canais para o usuário, em segundo plano ou abertos manualmente pelo usuário, taxas envolvidas
  • troca integrada entre onchain/LN, então não há necessidade de usar serviços externos
  • Breez também tem uma seção PoS integrada que pode ser facilmente usada por um pequeno comerciante
  • Guia Phoenix DC | Guia Green DC

Cenário de caso de uso

  1. o comerciante/usuário instala o aplicativo e salva a semente em um lugar seguro
  2. pode começar a receber pagamentos no LN imediatamente, clicando no botão receber e colocando o valor, criando o QR e a fatura do LN
  3. É melhor começar com um grande pagamento recebido para abrir um grande canal (como 10 milhões de sats) para você e pagar apenas uma vez a taxa de abertura. Se você começar com pequenas quantias, cada vez que receber um pequeno pagamento, estará abrindo um novo canal e cobrando uma taxa por esse canal.
  4. Depois de receber um grande pagamento, envie desse canal cerca de 90-95% dos fundos para outra carteira LN ou troque para outra carteira onchain (ou usando serviços de swap externos), deixando o canal com apenas uma pequena quantia de sats para mantê-lo aberto.
  5. Depois que o canal estiver quase vazio, você poderá começar a receber novamente no mesmo canal sem mais taxas de abertura.

18 - Soluções avançadas #

Descrição / recursos

Esta é uma solução complexa que pode ser executada em seu próprio hardware, de forma totalmente autocustodial, ou usando servidores hospedados / VPS.

A ideia é executar você mesmo toda a solução de pagamento, mas implica também um gerenciamento completo de financiamento, canais LN, liquidez, swaps etc.

  • Execute um nó Bitcoin LN dedicado com várias ferramentas de gerenciamento

  • Execute suas próprias instâncias de Servidor BTCPay ou Servidor LNbits

    Ambas essas "soluções completas" podem ser executadas em seu hardware local ou em servidores remotos

    Confira em: Voltage Cloud ou Nodana, soluções de hospedagem simples, onde você pode ativar um servidor BTCPay ou LNbits em questão de minutos.

    Na minha opinião honesta, o mais fácil e simples poderia estar no Nodana, usando um servidor phoenixd + instância LNbits. O Phoenixd não precisa de gerenciamento de canais complicado, é exatamente como uma carteira móvel Phoenix, mas usado como servidor de fonte de financiamento. E o LNBits é uma ferramenta poderosa com várias opções para comerciantes, incluindo um plugin simples WP Woocommerce.
