Welche Bitcoin LN-Lösungen könnten Sie als kleiner Händler nutzen?

Ich habe oft diese Frage gehört: „Wie kann ich als kleiner Händler anfangen, Bitcoin zu akzeptieren?“

Und hier möchte ich einige einfache, schnelle Lösungen, Testgelände und kleine, einfache Schritte für neue Benutzer vorstellen, die noch keine vollständige LN-Knotenmaschine betreiben können/wollen.

Zu erreichende Ziele:

  • Händler möchten anfangen, BTC über LN zu akzeptieren, mit einfachen Schritten, gerade genug, um es zu testen und zu lernen, wie es funktioniert und Sie können BTC empfangen.
  • Möglichst niedrige Kosten
  • Erfahren Sie mehr über die Bedingungen von Bitcoin und Lightning Network, wie Sie Wallets, QR-Codes, LN-Rechnungen, LNURL, Gebühren usw. verwenden
  • Eine Möglichkeit, die erhaltenen Gelder in ein „Hodling“-Wallet zu verschieben
  • Bereiten Sie sich mental auf den nächsten Schritt vor: den Betrieb eines Händler-LN-Knotens. Diese Lösungen sind nur der erste Schritt, vorübergehend, bis der Händler bereit ist, einen vollständigen Knoten und eine eigene Bank zur Verwaltung seines eigenen Geldes zu haben.
  • Zugriff auf ein WP Woocommerce-Plugin zur schnellen Integration in vorhandene E-Commerce-Webshops
  • Dies ist KEINE langfristige Lösung, sondern soll Sie nur vorbereiten und Ihnen zeigen, wie Bitcoin LN für einen kleinen Shop funktioniert. Für eine langfristige Lösung und Selbstverwaltung konsultieren Sie bitte den Abschnitt Erweiterte Lösungen weiter unten.

Verfügbare Lösungen:

Jede hat ihre eigenen Besonderheiten, Funktionen, Komplexitätsgrade und Anwendungsfälle. Ich schlage vor, mit 2-3 davon anzufangen und sich damit vertraut zu machen, während Sie andere studieren.

  • CoinOS – Depotverwaltung, sehr einfaches Onboarding, NWC, keine App erforderlich
  • Walletano – Depotverwaltung, sehr einfaches Onboarding, keine App erforderlich
  • IBEX Pay – Depotverwaltung, komplexes Händler-Dashboard, mehrere Funktionen, Fiat-Konvertierung, Woocommerce-Plugin
  • Blink Wallet – Depotverwaltung, LN-Adresse, PoS, API
  • Opago Pay – nicht Depotverwaltung, einfaches Offline-PoS, Fiat-Konvertierung, Woocommerce-Plugin
  • Swiss Bitcoin Pay – Depotverwaltung, einfache App, PoS-App, Woocommerce-Plugin
  • LifPay – Depotverwaltung, einfache App, PoS, NWC
  • OpenNode – Depotverwaltung für Zahlungsabwicklung, Woocommerce-Plugin
  • CoinGate – Depotverwaltung für Zahlungsabwicklung, Woocommerce-Plugin
  • Sats Mobi – Depotverwaltung, Telegram LN-Bot mit Optionen für PoS
  • LN Voltz Bank – Depotverwaltung, LNBits-Instanz für Ihren eigenen Gebrauch, Woocommerce-Plugin
  • Alby – selbstverwaltend, leistungsstarke Lösung mit mehreren Funktionen, NWC, Woocommerce-Plugin
  • Bluewallet - LNdhub-Wallet mit Verwahrung oder eingebetteter LDK-LN-Knoten mit Selbstverwahrung
  • Phoenix / Green / Breez / Electrum Mobile Wallet - einfache mobile selbstverwahrte LN-Knoten
  • Blixt - vollständig selbstverwahrter LND-Knoten auf Ihrem Mobiltelefon, LN-Adresse
  • Zeus - vollständig selbstverwahrter mobiler LND-Knoten, Remote-Knotenverwaltung, LNDHUB-Konten, LN-Adresse, PoS
  • Zaprite - Leistungsstarke Lösung mit mehreren gemischten Zahlungsmethoden
  • Erweiterte Lösungen Betreiben Sie Ihren eigenen BTC-LN-Knoten als Händler (BTCPay Server oder LNBits)

Ein besonders einfacher Fall könnte auch die Verwendung einer Lightning-Adresse sein. Für Freiberufler oder gelegentliche Rechnungen, die in BTC bezahlt werden, können Sie eine LN-Adresse mit einem föderierten Server oder Dienst oder sogar mit Ihrer eigenen Domäne einrichten, die mit Ihrem eigenen Knoten verknüpft ist. Hier ist eine Anleitung.

1 - CoinOS #

Beschreibung / Funktionen

  • https://coinos.io/
  • Weboberfläche, PWA, einfach zu verwenden auch auf Mobilgeräten, NWC (Nostr Wallet Connect)
  • Verwahrung, kein KYC
  • Sie müssen nichts installieren, erstellen Sie einfach ein anonymes Konto/Wallet
  • Nahtloser Wechsel zwischen Onchain / LN
  • Volle Kontrolle über Wallet-Schlüssel
  • Volle Unterstützung für LN-Adressen, jetzt können Sie eine als username@coinos.io haben
  • Volle Unterstützung für LNURL-Senden / -Empfangen / -Anmelden
  • Ecash-Unterstützung

Anwendungsfall Szenario

  1. Öffnen/erstellen Sie ein Konto und speichern Sie Benutzername/Passwort, 2FA, Schlüssel, LNURL-Authentifizierung
  2. Im Konto (oben rechts) sehen Sie, dass eine eindeutige Adresse für Ihr Wallet erstellt wurde, damit Sie leicht darauf zugreifen können, z. B. https://coinos.io/Benutzername
  3. Um Zahlungen zu empfangen, klicken Sie einfach auf die Schaltfläche „Empfangen“. Standardmäßig wird eine leere Lightning-Rechnung angezeigt. Sie müssen einen Betrag hinzufügen, einige LN-Wallets können noch keine leere LN-Rechnung bezahlen.
  4. Sie können auch die bereitgestellte LNURL verwenden, um Zahlungen ohne LN-Rechnung zu empfangen, sodass Sie diesen LNURL-QR-Code auf Papier ausdrucken und den Kunden vorlegen können, ohne das Online-Wallet öffnen zu müssen. Sobald der Kunde Ihnen die erfolgte Zahlung anzeigt, kann die Zahlung nicht mehr rückgängig gemacht werden, sodass keine weiteren Bestätigungen erforderlich sind.
  5. Es besteht auch die Möglichkeit, an eine Lightning-Adresse zu empfangen. Viele LN-Wallets können an username@coinos.io senden, und Sie müssen keine Rechnung erstellen. Der Kunde sendet lediglich den erforderlichen Betrag.
  6. Am Ende des Tages oder der Woche oder wann immer gewünscht, kann der Händler die Mittel vom CoinOS-Wallet auf ein beliebiges anderes persönliches Wallet abheben. Der Umtausch von LN-Geldern in die Onchain-Adresse erfolgt automatisch (es fällt eine Umtauschgebühr an). Es kann aber auch an ein anderes LN-Wallet gesendet werden, ohne dass eine andere Gebühr als die (extrem geringe) Tx-Gebühr anfällt.
  7. Wenn Sie ein einfaches Symbol auf dem Telefon-Desktop haben möchten, wie eine App, öffnen Sie https://coinos.io/username in Ihrem mobilen Browser (ich habe es mit Firefox auf Android getestet), gehen Sie zum Menü und klicken Sie auf „Installieren“. Es wird eine einfache Symbol-App erstellt, die Sie auf das Desktop-Widget Ihres Telefons ziehen und beim nächsten Mal wie eine App öffnen können.

10 - Sats Mobi

Beschreibung/Funktionen

  • https://t.me/satsmobi
  • Telegram LN Bot Wallet, kann in der Telegram-App verwendet und auch wie gewohnt in Zeus, BitBanana oder Bluewallet importiert werden LN-Wallet
  • Verwahrt, kein KYC
  • Dieses Wallet ist verwahrt und sollte daher nur für vorübergehende kleine Beträge verwendet und regelmäßig auf ein persönliches Wallet überwiesen werden
  • Unterstützung für LNURL und Lightning-Adresse (TGuser@sats.mobi), einfach im Offline-Modus zu verwenden
  • Kann auch als einfaches PoS verwendet werden
  • Sehr geringe Gebühren für Transaktionen, keine Gebühren für das Eröffnen/Aufrechterhalten des Konto-Wallets
  • BTC in der App kaufen

Anwendungsszenario

  1. Der Händler wird einen Telegram-Benutzernamen eröffnen/erstellen
  2. Es wird automatisch eine LNURL unter https://sats.mobi/@TG-user-name verfügbar sein, die empfangen kann sofortige LN-Zahlungen, ohne dass eine LN-Rechnung erforderlich ist, mindestens 100 Sats
  3. Es wird automatisch eine Lightning-Adresse als Benutzername@ln.tips verfügbar sein, an der Kunden direkt mit Wallets bezahlen können, die dies unterstützen.
  4. Händler können auch LN-Rechnungen erstellen, in Telegram im Direktchat mit @SatsMobiBot mit dem Befehl /invoice {Betrag Sats} {optionale Beschreibung} und dem Kunden den QR- oder LN-Rechnungscode anzeigen
  5. Händler können das SatsMobiBot-Wallet auch in die Zeus-/Bluewallet-App importieren und es als normales LN-Wallet verwenden und Rechnungen für seine Kunden erstellen, Gelder auf andere Wallets verschieben, auf Onchain umtauschen (mithilfe von Swap-Diensten)

11 - LN Voltz

Beschreibung/Funktionen

  • LN Voltz Bank
  • Ist eine öffentliche LNbits-Instanz, mit der sich leicht Depotkonten erstellen lassen, falls Sie schnell ein LNbits mit all seinen Funktionen benötigen.
  • Depot, kein KYC. Dieses Wallet ist verwahrt und sollte daher nur für vorübergehende kleine Beträge verwendet und regelmäßig auf ein persönliches Wallet überwiesen werden.
  • Unterstützung für LNURL und Lightning Address, PoS, Offline-PoS, mehrere nützliche Erweiterungen, Woocommerce-Plugin für LNbits
  • Hier ist eine vollständige Anleitung zur Verwendung von LNbits für Händler
  • sehr geringe Gebühren für Transaktionen, keine Gebühren für das Eröffnen/Aufrechterhalten des Konto-Wallets

Anwendungsszenario

  1. Händler eröffnet/erstellt kostenlos ein Voltz-Konto
  2. Kann seinen E-Commerce-Shop direkt mit dieser LNbits-Instanz verbinden, über Woocommerce LNbits-Plugin
  3. Zuerst müssen Sie die erforderlichen Erweiterungen hinzufügen: PayLinks, SatsPay Server, LNDHub, Scrub, Onchain, Boltz Swap, TPOS und andere, falls erforderlich.
  4. Erstellen Sie dann einen SatsPay-Link, um ihn über Woocommerce-Plugin-Gebühren in Ihrem E-Shop verwalten zu können.
  5. Fügen Sie ein Onchain-Xpub von einem Ihrer externen Wallets hinzu, das kalt oder heiß sein kann. Dies wird verwendet, um die Sats direkt in Ihr externes, selbstverwaltetes Wallet abzuheben.
  6. In der PayLinks-Erweiterung können Sie eine unbegrenzte Anzahl an LNURL- und LN-Adressen erstellen, die Sie für direkte statische QR-Zahlungscodes oder mit einem bestimmten Betrag verwenden können.
  7. Die Scrub-Erweiterung kann verwendet werden, wenn Sie die in Ihrem E-Shop berechneten Sats automatisch aus diesem verwahrten LNbits abheben möchten. Optional können Sie die Boltz-Erweiterung verwenden, um die Sats von LN auf eine beliebige Onchain-Adresse zu übertragen.
  8. Mit der LNdhub-Erweiterung können Sie den Zugriff auf Ihre Konten (in diesem Fall können Sie mehrere Wallet-Konten erstellen) mithilfe einer externen App wie Zeus, Bluewallet oder BitBanana verwalten.

14 - Phoenix, Green, Breez oder Electrum #

Beschreibung / Funktionen

  • Links: Phoenix | Green | Breez | Electrum
  • einfache mobile Geldbörsen, die Onchain/LN auf derselben Schnittstelle betreiben
  • öffnen Kanäle für den Benutzer, im Hintergrund oder manuell vom Benutzer geöffnet, Gebühren fallen an
  • integrierter Swap zwischen Onchain/LN, daher keine Notwendigkeit, externe Dienste zu verwenden
  • Breez hat auch einen integrierten PoS-Bereich, der von einem kleinen Händler problemlos verwendet werden kann
  • Phoenix DC-Leitfaden | Green DC-Leitfaden

Anwendungsszenario

  1. Händler/Benutzer installieren die App und speichern den Seed an einem sicheren Ort
  2. Sie können sofort mit dem Empfang von Zahlungen auf LN beginnen, indem Sie auf die Schaltfläche „Empfangen“ klicken und den Betrag eingeben, wodurch die QR- und LN-Rechnung erstellt wird
  3. Es ist besser, mit einem hohen Zahlungseingang zu beginnen, um einen großen Kanal (z. B. 10 Mio. Sats) für Sie zu öffnen und nur einmal die Eröffnungsgebühr zu zahlen. Wenn Sie mit kleinen Beträgen beginnen, wird bei jedem Erhalt einer kleinen Zahlung ein neuer Kanal geöffnet und Ihnen wird eine Gebühr für diesen Kanal berechnet.
  4. Sobald Sie eine große Zahlung erhalten, senden Sie von diesem Kanal etwa 90–95 % der Mittel an eine andere LN-Wallet oder tauschen Sie sie gegen eine andere Onchain-Wallet (oder verwenden Sie externe Swap-Dienste), sodass nur noch eine kleine Menge an Sats übrig bleibt, um den Kanal offen zu halten.
  5. Sobald der Kanal fast leer ist, können Sie wieder mit dem Empfang im selben Kanal beginnen, ohne weitere Eröffnungsgebühren.

18 - Erweiterte Lösungen #

Beschreibung/Funktionen

Dies ist eine komplexe Lösung, die auf Ihrer eigenen Hardware, vollständig selbstverwaltet oder mithilfe gehosteter Server/VPS ausgeführt werden kann.

Die Idee besteht darin, die gesamte Zahlungslösung selbst auszuführen, aber auch ein vollständiges Finanzierungsmanagement, LN-Kanäle, Liquidität, Swaps usw. einzubeziehen.

  • Betreiben Sie einen dedizierten Bitcoin-LN-Knoten mit mehreren Verwaltungstools

  • Betreiben Sie Ihre eigenen Instanzen von BTCPay Server oder LNbits Server

    Beide dieser „Volllösungen“ können auf Ihrer lokalen Hardware oder auf Remote-Servern ausgeführt werden

    Schauen Sie sich an: Voltage Cloud oder Nodana, einfache Hosting-Lösungen, mit denen Sie in wenigen Minuten einen BTCPay Server oder LNbits hochfahren können.

    Meiner ehrlichen Meinung nach könnte Nodana mit einem Phoenixd-Server + LNbits-Instanz am einfachsten und unkompliziertesten sein. Phoenixd benötigt keine komplizierte Kanalverwaltung, ist genau wie eine mobile Phoenix-Geldbörse, wird aber als Server für Finanzierungsquellen verwendet. Und LNBits ist ein leistungsstarkes Tool mit mehreren Optionen für Händler, einschließlich eines einfachen WP Woocommerce-Plugins.

    LNbits unterstützt mehrere Finanzierungsquellen als LND/CLN/phoenixd-Knoten, andere LNbits-Instanz, LN Pay, OpenNode, ZBD usw.

  • Betreiben Sie Ihre eigene private LN-Bank
  • Akzeptieren Sie Bitcoin- und Lightning-Zahlungen in Ihrem Online-Shop
  • Lassen Sie sich Zahlungen direkt an Ihre Cold Storage- oder Lightning-Adresse senden
  • Offline-/Online-PoS, Hardware-Tools, Geldautomaten, NFC-Zahlungen
