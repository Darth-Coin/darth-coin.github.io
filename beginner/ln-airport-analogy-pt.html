Uma analogia poderosa para explicar como funciona a rede Bitcoin Lightning

| | | | | | | | |

Este artigo é uma adaptação de Danny Scott (CoinCorner) Tópico do Twitter.

Postado originalmente em Substack em 10 de dezembro de 2022

ASSIMILAÇÕES

Aeroporto = nó BTC/LN

Rota = Canal Lightning

Aeronave = Capacidade do Canal

Passageiros = bitcoins / sats

Vamos imaginar que o aeroporto é nosso nó BTC/LN, é o hub que "controla" as rotas, os aviões e os passageiros.

CANAIS

Quando o aeroporto quer criar uma nova rota, por exemplo da Ilha de Man para Amsterdã, os aeroportos se comunicam e concluem um acordo (contrato) para essa rota. É basicamente o "contrato inteligente" parte do Bitcoin que cria um Canal Lightning.

CAPACIDADE DO CANAL

Como parte deste contrato, os aeroportos concordam com o tamanho da aeronave que fornecerá a conexão entre os dois. Algumas linhas terão aviões que podem transportar 50 passageiros, outras poderão transportar 200.

ROTAÇÃO

Os aeroportos não precisam estar conectados diretamente para permitir que um passageiro viaje de um destino para outro. Não posso voar diretamente da Ilha de Man para Nova York, mas como passageiro posso voar da Ilha de Man para Londres e depois de Londres para Nova York. O mundo pode ser conectado por estradas diferentes. É o mesmo quando você paga na Lightning Network.

LIQUIDEZ

Digamos que estamos voando da Ilha de Man para Londres e o avião pode transportar 50 passageiros (cada passageiro é um BTC), o que significa que a Ilha de Man tem capacidade para 50 voos de saída e Londres tem capacidade para 50 voos de entrada.

O avião sai da Ilha de Man e pousa em Londres com 50 passageiros.

O avião está agora no Aeroporto de Londres com 50 assentos disponíveis para retornar à Ilha de Man. Londres, portanto, agora tem uma capacidade de 50 lugares de ida e a Ilha de Man uma capacidade de 50 lugares de volta.

PAGAMENTOS DE VÁRIOS CAMINHOS

Esta analogia até ilustra como os pagamentos de vários caminhos funcionam.

50 passageiros desejam viajar de Nova York para Tóquio.

Eles não podem, no entanto, ir diretamente para lá porque não há estrada disponível, mas 30 deles podem pegar um voo de Nova York para Londres e depois para Tóquio. Os 20 restantes voam de Nova York para Barcelona e depois para Tóquio.

As 50 pessoas partiram de Nova York e chegaram a Tóquio, mas simplesmente pegaram rotas diferentes.

FECHAMENTO DE CANAIS

Então é hora de encerrar uma rota para o aeroporto, eles decidem que um último voo pode chegar para 50 passageiros. Os passageiros então deixam o aeroporto pela última vez por esta rota.

É como se o BTC deixasse a rede Lightning e retornasse para a rede Bitcoin (on-chain), como passageiros que retornam ao mundo exterior assim que saem do aeroporto.

CONCLUSÃO

Espero que isso ajude você a entender como a Lightning Network funciona, usando um exemplo da vida real.
